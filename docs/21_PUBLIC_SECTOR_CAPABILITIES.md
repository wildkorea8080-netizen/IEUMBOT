# IEUMBOT 공공기관 특화 역량 (제안·홍보용 요약)

> IEUMBOT는 **공공기관 특화 근거기반 AI 상담 챗봇**입니다. 등록된 공식 자료·법령·FAQ에
> 근거해 답변하고, 개인정보 보호·웹접근성·감사추적 등 공공 납품 요건을 기본 제공합니다.
> (행정안전부/NIA "지방세 AI 상담 서비스" 제안요청서 요구사항 분석 기반)

## 1. 공공 납품 필수요건 충족

| 요건 | 제공 기능 | 상태 |
|---|---|---|
| **개인정보 보호** | 질의 내 개인정보(주민·외국인등록번호, 전화, 이메일, 카드, 여권, 운전면허, 생년월일) **자동 탐지→마스킹** 후 답변. 마스킹본으로 LLM 전송·대화로그 저장 → **원본 미보관**. 관리자에서 마스킹/차단 방식 선택 | ✅ |
| **웹접근성(KWCAG)** | 임베드 위젯: `role=dialog`·제목 연결, 새 답변 스크린리더 낭독(`aria-live`), 입력 라벨, 키보드 조작(Esc 닫기·포커스 복귀), 포커스 표시, 명도대비 4.5:1 준수 | ✅ |
| **근거 제시(환각 최소화)** | RAG 기반, 답변마다 참조 문서·조문·페이지·**유사도 점수**·원문 링크 표시. 관리자 대화상세에서 **실제 사용 프롬프트**까지 열람(감사·검증) | ✅ |
| **감사·로그** | 관리자 활동 감사로그, 세션 단위 대화 로그 보존·조회 | ✅ |
| **보안/가드레일** | 비정상 접근·부적절 발언 탐지, 가드레일 규칙, 에스컬레이션 | ✅ |

## 2. 공공 차별화 기능 (경쟁 우위)

- **법령/공식 OpenAPI 자동 현행화** — 국가법령정보센터 등 공식 API를 등록하면 주기적으로
  수집·색인. **법령 개정 자동 반영**. 등록 전 "테스트 호출" 미리보기 제공.
- **품질 개선 루프** — 미답변/오답을 큐로 수집 → 관리자가 **"FAQ로 등록"** 원클릭으로
  지식화 + 자동 해결 → 다음부터 자동 응답(지속 학습).
- **상담 주제 분포 대시보드** — 사용자 질문을 키워드로 집계해 자주 묻는 주제·답변 결과
  분포를 시각화 → 정책·서비스 개선 근거.
- **조건별 안내(관련 정보)** — 답변에 맞춰 전화/방문/온라인 상담 링크·파일 등을 자동 첨부.
- **근거 신뢰성 감사 UI** — "왜 이렇게 답했는지"를 근거·점수·프롬프트로 추적 가능.

## 3. 기본 플랫폼 강점

- RAG(키워드+벡터) 검색 + FAQ 시맨틱 매칭, 임계값·출처 게이트
- 문서(PDF/HWP/DOCX/XLSX)·웹사이트·텍스트·**API** 등 다양한 지식 소스 + 주기 동기화
- Shadow DOM 임베드 위젯(호스트 CSS 격리), 배너/리치 카드·아이콘·링크, 마크다운 표/목록 렌더
- 멀티테넌트(기관·챗봇 분리), 런타임 모델 설정, 답변 캐시/워커/관측성(선택)

## 4. 로드맵 (공공 심화 — 별도 구축 항목)

| 항목 | 가치 | 규모 |
|---|---|---|
| 음성(STT/TTS)·다국어 | 디지털 취약계층(고령·장애·외국인) 접근성 | 大(외부 음성 API 연동) |
| 대민/내부직원 서비스 분리 + 역할기반 접근통제(RBAC) | 공무원 업무지원 에이전트 분리 운영 | 中~大 |
| AI 에이전트 라우팅(법령검색·통계·요약 도구 호출, MCP) | 복합 질의 자동 처리 | 大 |
| 기능별 모델 선택 + 토큰·응답시간 관측 대시보드 | 대민=민간LLM/내부=공공LLM 등 | 中 |
| 국산 LLM·공공클라우드·망분리 대응 | 행정망/공공클라우드 배포 | 大(인프라) |
| 명도대비 외 디자인 접근성 심화, 개인정보 영향평가(PIA) 자료 | 인증·심사 대응 | 中 |
