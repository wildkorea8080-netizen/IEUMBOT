# 16_SECURITY_PRIVACY_SPEC.md

## 1. 원칙
- 공공서비스 안전성 우선
- 최소 수집/최소 노출
- 추적 가능 감사 로그

## 2. 인증/인가(MVP)
- 관리자: 이메일/비밀번호 + 액세스 토큰
- 보호 API는 Bearer 토큰 필수
- 미인증 401, 권한 부족 403

## 3. 비밀번호 보안
- bcrypt/argon2 해시 저장
- 평문 저장 금지
- 로그인 실패 응답은 계정 존재 여부를 노출하지 않음

## 4. 프롬프트 인젝션 대응
- 문서 텍스트를 명령이 아닌 근거 데이터로 처리
- 시스템 지시 무시 요청 차단
- 외부검색 결과는 화이트리스트 도메인만 허용

## 5. 공공서비스 안전 응답 규칙
- 근거 부족 시 생성 차단
- 추측 금지 주제:
  - 자격/수급 가능성
  - 법률 해석
  - 지원금액
  - 마감일
  - 결과 예측

## 6. 개인정보 처리
- 채팅 로그 저장 시 마스킹 적용
- 민감정보 원문 재출력 금지
- 관리자 UI에서도 마스킹값 우선 노출

## 7. 로그 보안
- requestId 기반 추적
- 실패 사유/근거 부족 사유/추측 금지 트리거 기록
- 토큰/비밀번호/원문 민감정보 로그 금지

## 8. 저장/전송 보호
- DB 연결 TLS 권장
- 운영환경 HTTPS 강제
- 비밀값은 env/secret manager로만 주입

## 9. 사고 대응
- 잘못된 공공안내 탐지 시 해당 챗봇 즉시 제한 모드
- 공지/문서 재색인 후 재오픈
- 사고 로그와 재발방지 조치 기록
