# 19_TEST_PLAN.md

## 1. 테스트 목표
공공기관 챗봇 품질 기준(근거 우선/안전 폴백/추측 금지)을 MVP에서 검증한다.

## 2. 테스트 범위
- 프론트: 로그인/세션/가드
- 백엔드: 인증 API, 보호 라우트, 에러 코드
- RAG: 하이브리드 검색, 근거충족도 검증, 템플릿 응답
- 운영: 정기 색인, 로그 품질 지표

## 3. 핵심 테스트 항목

### 3.1 우선순위 검증
- 문서와 웹소스가 동시에 존재할 때 문서 근거가 우선 채택되는지 확인
- 외부검색은 예외 조건에서만 실행되는지 확인

### 3.2 전처리/메타데이터 검증
- 버전/시행일/부서/대상/예외/페이지/섹션 누락 여부 점검

### 3.3 질의 전처리 검증
- 오탈자 보정 정확도
- 민원형 질문 재작성 품질
- 복합질문 분해 일관성

### 3.4 증거 충족도 검증
- 임계치 미달 시 생성 차단
- 폴백 템플릿 반환 여부

### 3.5 답변 템플릿 검증
- 결론/근거/상세 안내/출처/주의 5구역 누락 검사

### 3.6 추측 금지 검증
아래 질의군에 대해 단정형 답변 금지:
- 수급 가능 여부
- 법 해석
- 금액 확정
- 마감일 확정
- 결과 예측

### 3.7 인증 흐름 검증
- 로그인 성공/실패
- 만료 토큰 접근 시 로그인 리다이렉트
- 로그아웃 후 보호 페이지 접근 차단

## 4. 샘플 테스트 케이스
- TC-QA-01: 문서 근거 충분 질의 → 근거 포함 정상 응답
- TC-QA-02: 근거 부족 질의 → fallback.insufficientEvidence 반환
- TC-QA-03: eligibility 질의(근거 부족) → 추측 금지 + 담당부서 안내
- TC-QA-04: 최신 공지 반영 직후 질의 → 공지 코퍼스 인용 확인
- TC-AUTH-01: 잘못된 비밀번호 → 401 + 에러 메시지
- TC-AUTH-02: 만료 토큰으로 `/admin/*` 접근 → 401

## 5. 로그 기반 개선 검증
- 주간 리포트 항목:
  - fallback 비율
  - noGuess 트리거 비율
  - 미회수 질의 Top N
  - 재색인 필요 소스 목록
