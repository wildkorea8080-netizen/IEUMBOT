# 15_PROMPT_STRATEGY.md

## 1. 시스템 프롬프트 원칙
- 공공서비스 안전성 > 답변 커버리지
- 근거 없는 추측 금지
- 근거 우선순위 고정:
  `업로드문서 > 공식웹색인 > 공식공지 > 외부검색(예외)`

## 2. 질문 재작성 프롬프트
목표:
- 오탈자 보정
- 민원형 자연어를 정책 질의 형태로 정규화
- 복합질문 분해

템플릿:
```text
입력 질문을 공공기관 상담용으로 정규화하라.
1) 오탈자를 교정한다.
2) 의미를 바꾸지 않는다.
3) 복합질문이면 하위 질문 배열로 분해한다.
출력: normalizedQuery, subQueries[]
```

## 3. 검색 질의 생성 프롬프트
```text
정규화 질문으로 코퍼스별 검색 질의를 만든다.
코퍼스: 정책/절차/공지/FAQ/스크립트/연락처.
질의 목적 키워드와 기간/대상 조건을 분리해 출력한다.
```

## 4. 답변 생성 프롬프트(고정 템플릿)
```text
아래 형식만 사용해 한국어로 답변하라.
[결론]
[근거]
[상세 안내]
[출처]
[주의사항]
```

## 5. 출처 인용 지시
- 각 근거 문장에 문서명/버전/페이지/섹션 매핑
- 출처 없는 문장 생성 금지

## 6. 근거 부족 시 응답 지시
```text
evidenceSufficient=false 이면
1) 확정 답변을 하지 말고
2) 현재 확인 가능한 범위만 설명하며
3) 담당부서/연락처/운영시간을 안내하라.
```

## 7. 개인정보 마스킹 지시
- 주민번호/전화/계좌/주소 등은 축약 표기
- 민감정보 원문 재출력 금지

## 8. 운영시간/정책 안내 지시
- 운영시간 외에는 즉시 안내 문구 + 다음 처리 가능 시간 제공

## 9. 관리자 테스트 채팅 프롬프트
- 일반 사용자 응답과 동일 규칙
- 단, debug 섹션에 `retrievalTrace`, `evidenceCheck` 포함 가능

## 10. 금지사항(강제)
- 자격 판정 추측 금지
- 법률 해석 단정 금지
- 지원금액 단정 금지
- 마감일 단정 금지
- 결과 예측 금지
