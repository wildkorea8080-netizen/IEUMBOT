# 20_ROADMAP_MVP.md

## 1. MVP 범위
- 관리자 인증
- 문서/공식웹 정기 색인
- 하이브리드 검색
- 근거 충족도 검증
- 공공서비스 고정 답변 템플릿
- 안전 폴백/에스컬레이션
- 로그 기반 품질 개선

## 2. 단계별 계획

### Phase 1 (주 1~2): 품질 정책 고정
- 답변 우선순위 정책 확정
- 추측 금지 정책 확정
- 코퍼스 분리 구조 확정

### Phase 2 (주 3~4): 데이터/색인 기반
- 문서 메타데이터 확장
- 공식웹 정기 색인 파이프라인
- 공지 코퍼스 우선 처리

### Phase 3 (주 5~6): 검색/생성 품질
- 하이브리드 검색 도입
- 질의 정규화/분해 도입
- evidence sufficiency gate 적용

### Phase 4 (주 7): 위젯/콘솔 통합
- 템플릿 응답 UI 반영
- 폴백/에스컬레이션 UX 반영
- 관리자 테스트 채팅 디버그 표시

### Phase 5 (주 8): 검증/운영 준비
- 품질 테스트 시나리오 수행
- 로그 대시보드 운영지표 검증
- 릴리스 체크리스트 완료

## 3. 우선순위
1) 안전성 정책  
2) 근거 품질  
3) 최신성(정기 색인)  
4) UX 완성도

## 4. 의존성
- 공식 문서/공지 확보
- 공식 도메인 화이트리스트 확정
- 부서 연락처/운영시간 데이터 확보

## 5. 위험요소 및 대응
- 위험: 근거 부족 질의 다발  
  대응: 폴백 템플릿 + 로그 기반 재색인
- 위험: 공지 반영 지연  
  대응: 공지 코퍼스 색인 주기 단축
- 위험: 금지 주제 추측 응답  
  대응: no-guess 룰 + 테스트 게이트

## 6. 완료 기준(DoD)
- 우선순위 정책이 검색 trace에 반영됨
- 근거 부족 시 생성 차단 100%
- 금지 주제 단정 응답 0건
- 관리자 로그에서 품질지표 조회 가능
