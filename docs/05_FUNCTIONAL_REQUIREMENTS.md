# 05_FUNCTIONAL_REQUIREMENTS.md

본 문서는 기존 기능 요구사항 중 공공기관 품질 강화를 위해 반드시 반영할 항목만 정리한다.

## 공통 정책
- API/프론트 외부 계약: `camelCase`
- 백엔드 내부/DB: `snake_case`
- 우선순위 표기: Must / Should / Could

## 품질 강화 기능 요구사항(보완)

| ID | 제목 | 설명 | 우선순위 | 수용 기준 |
|---|---|---|---|---|
| FR-QA-001 | 답변 근거 우선순위 고정 | 문서 > 공식웹색인 > 최신공지 > 외부검색(예외) 순서 강제 | Must | 검색 trace에 우선순위 단계가 기록되고 역순 사용 금지 |
| FR-QA-002 | 안전성 우선 정책 | 근거 부족 시 답변 생성보다 폴백/상담연결 우선 | Must | 근거 임계치 미달 요청은 생성 API가 `insufficientEvidence`로 종료 |
| FR-QA-003 | 문서 구조 보존 전처리 | 시행일, 부서, 대상, 예외, 버전, 섹션, 페이지 메타데이터 유지 | Must | 청크 저장 시 필수 메타데이터 누락률 0% |
| FR-QA-004 | 하이브리드 검색 | BM25 + 벡터 결합, 최소 상위 N개 증거 병합 | Must | 벡터 단독 모드 비활성, 검색 결과에 lexicalScore/vectorScore 포함 |
| FR-QA-005 | 질문 정규화 | 오탈자 보정, 민원형 재작성, 복합질문 분해 | Must | 원문/정규화문/분해질문이 로그에 저장 |
| FR-QA-006 | 증거 충족도 검증 | 생성 전 증거 커버리지/충돌 여부/최신성 검사 | Must | 검증 실패 시 템플릿 폴백 반환 |
| FR-QA-007 | 공공서비스 고정 답변 템플릿 | 결론→근거→상세안내→출처→주의 문단 고정 | Must | 템플릿 필드 누락 시 응답 실패 처리 |
| FR-QA-008 | 추측 금지 주제 차단 | 자격, 법 해석, 지원금액, 마감일, 결과예측 추측 응답 금지 | Must | 금지 주제는 확정 표현 차단 및 확인 경로 제시 |
| FR-QA-009 | 공식웹 정기 색인 | 실시간 스크래핑 금지, 스케줄 기반 수집/재색인 | Must | 주기 실행 로그와 마지막 성공 시각 제공 |
| FR-QA-010 | 코퍼스 도메인 분리 | 정책/절차/공지/FAQ/스크립트/연락처 코퍼스 분리 | Must | 관리자 화면에서 도메인별 활성화/비활성화 가능 |
| FR-QA-011 | 근거 부족 에스컬레이션 | 담당부서/연락처/운영시간 안내 포함 폴백 | Must | 폴백 응답에 상담 경로 필수 포함 |
| FR-QA-012 | 로그 기반 품질개선 | 미회수 질의, 폴백 사유, 금지주제 탐지율 집계 | Must | 주간 리포트 생성 가능 |

## 인증/관리 기능 보완
| ID | 제목 | 설명 | 우선순위 | 수용 기준 |
|---|---|---|---|---|
| FR-AUTH-001 | 관리자 로그인 | 이메일/비밀번호 인증 및 액세스 토큰 발급 | Must | 실패 시 401, 성공 시 만료시각 포함 |
| FR-AUTH-002 | 관리자 보호 라우트 | 관리자 API는 인증 토큰 필수 | Must | 미인증 요청 401 |
| FR-AUTH-003 | 로그아웃 | 토큰 무효 처리(클라이언트 삭제 + 서버 로그) | Must | 로그아웃 후 보호 API 접근 불가 |
