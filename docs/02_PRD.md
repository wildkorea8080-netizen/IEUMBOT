# 02_PRD.md

## 1. 제품 개요
이음봇(IEUMBOT)은 공공기관 웹사이트에 삽입되는 문서 기반 상담 챗봇이다.  
MVP 범위는 `관리자 콘솔 + 문서/웹 인덱싱 + 위젯 질의응답 + 출처/로그`로 한정한다.

## 2. 문제 정의
- 답변 정확도보다 답변 생성량이 우선되면 공공서비스 리스크가 커진다.
- 최신 공지/공고 반영이 지연되면 안내 신뢰도가 급격히 하락한다.
- 문서 구조(시행일, 예외, 대상) 손실 시 eligibility/기한/금액 오답 가능성이 높다.

## 3. 목표 및 성공 기준
### 목표
- 공공기관 상담에서 **정답률보다 안전성 우선** 정책을 기본값으로 적용한다.
- 근거 부족 시 추측 답변 대신 명시적 한계 안내/상담 연결을 수행한다.

### 성공 기준(MVP)
- 근거 포함 응답 비율: 95% 이상
- 근거 부족 시 안전 폴백 수행률: 99% 이상
- 금지 주제(수급 가능성/법 해석/금액 확정/마감일 단정/결과 예측) 추측 응답률: 0%

## 4. 범위(In Scope)
- 고정 답변 우선순위:
  1) 업로드 공식 문서  
  2) 정기 색인된 공식 웹페이지  
  3) 최신 공식 공지/공고  
  4) 외부 웹검색(제한적 예외)
- 하이브리드 검색(BM25 + 벡터)
- 질문 정규화/오탈자 보정/민원형 재작성/복합질문 분해
- 근거 충족도 검증 후 답변 생성
- 공공서비스 답변 템플릿 강제

## 5. 제외 범위(Out of Scope)
- 소셜 로그인, 고급 RBAC, 실시간 외부 웹 스크래핑
- 외부 웹검색 기본 활성화
- 개인별 맞춤 결과 예측 모델

## 6. 핵심 사용자 시나리오
- 일반 사용자: 정책/절차/서류/운영시간 문의
- 관리자: 문서/웹소스 관리, 색인 상태 점검, 테스트 채팅, 로그 검토

## 7. 사용자 기능 요구사항
- 답변은 항상 결론/근거/안내/출처/주의 순서로 구성
- 근거 부족 시 “확인 불가 + 확인 경로 + 담당부서 안내” 필수
- 출처는 문서명 + 버전 + 페이지/섹션 포함

## 8. 관리자 기능 요구사항
- 코퍼스 분리 운영:
  - 정책/사업
  - 절차/서식
  - 공지/공고
  - FAQ
  - 내부 상담 스크립트
  - 연락처/운영시간
- 공식 웹소스 정기 색인(스케줄 기반), 수동 재색인
- 답변 우선순위/외부검색 예외정책 설정

## 9. 시스템 요구사항
- 증거 충족도 임계치 미달 시 생성 단계 차단
- 외부검색은 예외 규칙(공식 도메인만, 관리자 승인 상태)에서만 허용
- 로그 기반 품질 개선 루프(질문 유형, 미회수 원인, 폴백 사유, 재색인 필요 신호)

## 10. 제약사항
- 공공기관 운영 특성상 보수적 응답 정책 유지
- 최신성은 “정기 색인 주기 + 공지 우선 코퍼스”로 보장

## 11. MVP 정의
- 문서/공식웹 기반 하이브리드 검색
- 근거 부족 차단 + 안전 폴백
- 고정 답변 템플릿
- 관리자 품질 로그 대시보드(기초 지표)

## 12. 향후 확장 방향
- 정책 도메인별 세분화 랭킹
- 고도화된 재랭킹/품질평가 자동화
- 부서별 운영 정책 템플릿
