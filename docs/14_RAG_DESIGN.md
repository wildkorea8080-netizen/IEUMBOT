# 14_RAG_DESIGN.md

## 1. 목표
공공기관 상담에서 환각 최소화와 안전성 확보를 최우선으로 하는 RAG 파이프라인 설계.

## 2. 코퍼스 설계
분리 코퍼스:
1) 정책/사업  
2) 절차/서식  
3) 공지/공고  
4) FAQ  
5) 내부 상담 스크립트  
6) 연락처/운영시간

## 3. 수집 우선순위
1. 업로드 공식 문서
2. 정기 색인된 공식 웹콘텐츠
3. 최신 공식 공지/공고
4. 외부 검색(제한 예외)

## 4. 전처리(필수 보존 필드)
- 문서 구조: 제목/절/항/페이지
- 문서 메타: 버전, 시행일, 담당부서, 대상, 예외
- 인용 메타: pageNumber, sectionTitle, sourceType

## 5. 질의 전처리
- query normalization
- typo correction
- civic-question rewriting
- multi-part query decomposition

## 6. 검색 전략
- 하이브리드 검색: BM25 + Vector
- 도메인 코퍼스 라우팅 후 병합
- 최신 공지 코퍼스 가중치 상향

## 7. 증거 충족도 검증
생성 전 검증:
- 질의 요구사항 대비 근거 커버리지
- 근거 간 충돌 여부
- 최신성(시행일/공지일)

임계치 미달 시:
- 생성 차단
- 폴백/상담 연결 응답 생성

## 8. 생성 전략
- 고정 템플릿:
  1) 결론
  2) 근거
  3) 상세 안내
  4) 출처
  5) 주의사항
- 출처 누락 시 답변 실패 처리

## 9. 추측 금지 규칙
아래 항목은 근거 부족 시 확정 표현 금지:
- 수급/자격 판단
- 법률 해석
- 지원금액 단정
- 마감일 단정
- 결과 예측

## 10. 웹 색인 정책
- 실시간 스크래핑 금지
- 스케줄 기반 정기 색인
- 관리자 수동 재색인 지원

## 11. 로그 기반 개선
로그 필수 항목:
- normalizedQuery
- decompositionCount
- retrievedSourceType
- evidenceScore
- fallbackReason
- noGuessTriggered

운영 루프:
1) 주간 실패질문 집계
2) 누락 문서/공지 확인
3) 재색인/코퍼스 보강
4) 프롬프트/임계치 조정
